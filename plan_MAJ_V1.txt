=====================================================================
  PLAN : V1 Best-Effort Network Updates (MAJ)
=====================================================================

1. CONTEXTE
-----------
Le sujet demande une V1 ou chaque evenement local (spawn, deplacement,
attaque, mort) est immediatement envoye aux autres joueurs via le canal
C/UDP existant (partie_c/). A la reception, le joueur distant integre
la MAJ dans son affichage. Pas de propriete reseau en V1 - concurrence
sauvage, incoherences acceptees.


2. APPROCHE : SNAPSHOT-DIFF
----------------------------
Apres chaque tour, on compare l'etat courant des unites avec un
snapshot precedent. Seules les differences sont envoyees comme messages.
Cote reception, on poll les messages et on les applique a l'etat local.
Zero modification de la logique de jeu (map.py, unit.py, IAs).

5. FORMAT DES MESSAGES
-----------------------
Le protocole existant est: "id_joueur|pos_x|pos_y|action|target_id"

  SPAWN  -->  "0|1500|2000|2|R_K_0:K"
              (equipe 0=R, position 15.00/20.00, action SPAWN, id:type)

  MOVE   -->  "0|1650|2100|0|R_K_0"
              (equipe 0=R, nouvelle position 16.50/21.00, action MOVE, id)

  ATTACK -->  "1|3500|0|1|B_C_3"
              (equipe 1=B, nouveau HP=35.00, action ATTACK, id de la victime)

  DEATH  -->  "0|1500|2000|5|R_K_0"
              (equipe 0=R, derniere position, action DEATH, id)

Les positions float sont multipliees par 100 pour etre envoyees en int.

7. VERIFICATION
----------------
  1. Lancer le C relay: ./main.exe (sans argument = pas de peer)
  2. Lancer le jeu:   python main.py battle run stest1 Major_DAFT Major_DAFT -n
  3. Verifier que "[NET] Synchronisation reseau initialisee" s'affiche
  4. Le jeu doit tourner normalement (pas de regression)
  5. Test 2 joueurs: 2 relays C pointant l'un vers l'autre + 2 instances Python


8. LIMITATIONS V1
------------------
  - Pas de propriete reseau (sera en V2)
  - Incoherences possibles si 2 joueurs modifient la meme unite
  - Pas de garantie d'ordre des messages (UDP)
  - Pas de retransmission si perte de paquet
  - Observation en "best-effort" avec decalage temporel acceptable
